# AGENTS

## Product UI Principles

- In-app UI should not carry detailed usage instructions when that guidance is published separately on the web.
- Prefer operational surfaces over tutorial copy. Show state, structure, and controls first.
- Keep labels short and product-like. Avoid explanatory paragraphs unless they are required to prevent a real usability failure.
- For placement and canvas tools, prioritize spatial context, current selection state, and direct manipulation affordances over onboarding text.
- Multi-monitor views should reflect physical layout and aspect ratio accurately enough for users to recognize the target display at a glance.
